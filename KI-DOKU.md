# KI-DOKU

## Evaluation von KI-Werkzeugen

### Verglichene Tools

1. Codex
2. ChatGPT oder GitHub Copilot

### Kriterien für die Nutzwertanalyse

- Geschwindigkeit bei Codevorschlägen
- Qualität der Struktur über mehrere Dateien
- Hilfe bei Fehlersuche und Überarbeitung
- Verständlichkeit der Erklärungen
- Eignung für HTML, CSS und JavaScript ohne Frameworks

### Kurzbewertung

#### Codex

- Stark bei konkreten Projektanpassungen direkt im bestehenden Code
- Gut geeignet für mehrere zusammenhängende Änderungen in HTML, CSS und JavaScript
- Hilfreich bei der Prüfung, welche Anforderungen im Auftrag noch fehlen

#### ChatGPT oder GitHub Copilot

- Gut für Ideen, Textvorschläge und alternative Lösungsansätze
- Praktisch für schnelles Brainstorming oder einzelne Codebausteine
- Weniger praktisch, wenn eine komplette Projektstruktur ueber mehrere Dateien hinweg angepasst werden soll

### Begründung der Werkzeugwahl

Für dieses Projekt eignet sich Codex besonders gut, weil bestehende Dateien direkt überarbeitet und neue Seiten, Datenstrukturen und Formularlogik zusammenhängend ergänzt werden konnten. Dadurch liessen sich die Anforderungen aus dem Auftrag systematisch umsetzen.

## Einsatz der KI im Projekt

Die KI wurde in folgenden Bereichen eingesetzt:

- Analyse des bestehenden Projekts gegen den Projektauftrag
- Erweiterung des Webshops auf mindestens 10 Produkte
- Aufbau einer dynamischen Produktübersicht mit Kategoriefilter
- Erstellung einer Detailseite mit Bild, Beschreibung, Video und Bestellformular
- Erweiterung der Startseite um Kategorien und Newsletter-Anmeldung
- Erweiterung der Kontaktseite um Teaminformationen und Betreffauswahl
- Formulierung von Produkttexten und Doku-Inhalten

## Beispiele für die Nutzung

- Aus einer einfachen Produktseite wurde eine gefilterte Übersicht mit Kategorien wie Floral, Frisch, Holzig und Orientalisch aufgebaut.
- Die Produktdetails werden über einen URL-Parameter geladen, damit eine einzelne Detailseite für mehrere Produkte verwendet werden kann.
- Formulare für Newsletter, Kontakt und Bestellung wurden mit JavaScript ergänzt, damit eine direkte Rückmeldung auf der Seite erfolgt.

### Stärken der KI

- Schnelle Strukturierung von Anforderungen
- Gute Unterstützung bei Fleissarbeit in mehreren Dateien
- Hilfreich bei responsivem Layout und bei wiederholbaren Komponenten

### Schwächen der KI

- Ergebnisse müssen immer gegen den eigentlichen Auftrag geprüft werden
- Inhalte wie Produkttexte oder Doku müssen fachlich und sprachlich nachbearbeitet werden
- Ohne Kontrolle können wichtige Abgabe-Details wie Dateinamen oder Dokumentformate übersehen werden

### Erkenntnisse

Der KI-Einsatz spart Zeit, besonders bei der technischen Umsetzung und beim Erweitern bestehender Seiten. Trotzdem bleibt die eigene Kontrolle wichtig, damit Anforderungen, Inhalte und Abgabeformat wirklich stimmen.
